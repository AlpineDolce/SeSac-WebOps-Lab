## 2. 핵심 HTML 요소: 콘텐츠 구조화

### 2.1. 텍스트 관련 태그
HTML은 웹 페이지의 텍스트 콘텐츠에 구조와 의미를 부여하는 다양한 태그를 제공합니다. 이러한 태그들을 올바르게 사용하는 것은 웹 접근성(Accessibility), 검색 엔진 최적화(SEO), 그리고 코드의 유지보수성을 높이는 데 매우 중요합니다.

- **2.1.1. 제목 (Headings): `<h1>` ~ `<h6>`**
    *   **역할**: 문서나 섹션의 제목을 나타냅니다. `<h1>`이 가장 중요한 최상위 제목이며, `<h6>`으로 갈수록 중요도가 낮아집니다.
    *   **시맨틱 사용**: 제목 태그는 문서의 계층적 구조를 나타내기 위해 사용되어야 합니다. 예를 들어, `<h1>`은 페이지당 한 번만 사용하고, 그 다음으로 중요한 제목은 `<h2>`, 그 다음은 `<h3>` 순으로 사용합니다. 시각적인 크기 때문에 제목 레벨을 건너뛰는 것은 피해야 합니다 (예: `<h1>` 다음에 바로 `<h3>` 사용).
    *   **SEO 및 접근성**: 검색 엔진은 제목 태그를 사용하여 문서의 주요 주제와 구조를 파악합니다. 스크린 리더 사용자에게도 페이지의 논리적 구조를 이해하는 데 중요한 역할을 합니다.
    *   **코드 사례**:
        ```html
        <h1>웹 개발 기초</h1>
        <h2>HTML의 중요성</h2>
        <h3>시맨틱 HTML</h3>
        <h4>제목 태그의 올바른 사용</h4>
        ```

- **2.1.2. 문단 (Paragraph): `<p>`**
    *   **역할**: 하나의 독립적인 문단을 정의합니다. 브라우저는 `<p>` 태그의 위아래에 자동으로 약간의 여백(margin)을 추가하여 문단을 시각적으로 구분합니다.
    *   **주의**: `<p>` 태그 안에 다른 블록 레벨 요소(예: `<div>`, `<h1>`)를 중첩할 수 없습니다.
    *   **코드 사례**:
        ```html
        <p>
            HTML은 웹 페이지의 구조를 정의하는 마크업 언어입니다.
            모든 웹 콘텐츠의 뼈대가 됩니다.
        </p>
        <p>
            CSS는 HTML 요소의 스타일을 지정하고, JavaScript는 동적인 기능을 추가합니다.
        </p>
        ```

- **2.1.3. 줄바꿈 (`<br>`) 및 수평선 (`<hr>`)**
    *   **`<br>` (Line Break)**: 텍스트 내에서 강제로 줄을 바꿀 때 사용되는 빈 태그(empty tag)입니다. 문단 분리(`</p><p>`) 대신 단순히 줄만 바꿔야 할 때 (예: 시의 행, 주소의 각 줄) 사용합니다.
    *   **`<hr>` (Horizontal Rule)**: 내용의 주제가 변경되었음을 나타내기 위해 시각적인 수평선을 삽입하는 빈 태그입니다. 의미론적인 구분선 역할을 합니다.
    *   **코드 사례**:
        ```html
        <p>
            서울특별시 강남구 테헤란로 123<br>
            ABC 빌딩 10층
        </p>
        <hr>
        <p>새로운 섹션의 시작입니다.</p>
        ```

- **2.1.4. 텍스트 강조 및 의미 부여 태그**
    HTML은 텍스트의 특정 부분에 의미론적인 강조나 다른 뉘앙스를 부여하는 다양한 인라인 태그를 제공합니다.

    *   **`<strong>` vs `<b>`**:
        *   **`<strong>`**: 텍스트가 내용상 **매우 중요함**을 나타냅니다. (예: 경고, 핵심 정보). 스크린 리더는 이 부분을 강조하여 읽어주며, 브라우저는 기본적으로 굵은 글씨체로 표시합니다. **시맨틱한 중요성**이 있습니다.
        *   **`<b>`**: 특별한 의미론적 중요성 없이, 단순히 주변 텍스트와 구분하기 위해 **굵은 글씨체**로 표시할 때 사용합니다. (예: 제품명, 키워드, 기사 요약의 첫 문장). **시각적인 강조**에 가깝습니다.
    *   **`<em>` vs `<i>`**:
        *   **`<em>` (Emphasis)**: 텍스트의 특정 부분을 **강조**하여 억양을 다르게 표현하고 싶을 때 사용합니다. (예: "이것은 *정말* 중요합니다!"). 스크린 리더는 기울임 톤으로 읽어주며, 브라우저는 기울임꼴로 표시합니다. **시맨틱한 강조**입니다.
        *   **`<i>` (Italic)**: 기술 용어, 외국어 구문, 생각, 선박 이름 등 일반적인 산문과 다른 톤이나 분위기의 텍스트에 사용됩니다. 시각적으로는 기울임꼴이지만, `<em>`처럼 의미론적 강조는 없습니다. **다른 뉘앙스**를 표현합니다.
    *   **`<mark>`**: 텍스트의 특정 부분을 **하이라이트**하여 현재 문맥에서 관련성이 있거나 중요함을 나타냅니다. (예: 검색 결과에서 일치하는 단어).
    *   **`<small>`**: 저작권 표시, 법적 고지, 작은 글씨의 면책 조항 등 **부가적이거나 작은 글씨**로 표시될 내용을 나타냅니다.
    *   **`<del>` 및 `<ins>`**:
        *   **`<del>`**: 문서에서 **삭제된 텍스트**를 나타냅니다. (취소선으로 표시)
        *   **`<ins>`**: 문서에 **삽입된 텍스트**를 나타냅니다. (밑줄로 표시)
        *   주로 문서의 변경 이력을 보여줄 때 유용합니다.
    *   **`<sub>` 및 `<sup>`**:
        *   **`<sub>` (Subscript)**: 아래 첨자 텍스트를 나타냅니다. (예: 화학식 H<sub>2</sub>O)
        *   **`<sup>` (Superscript)**: 위 첨자 텍스트를 나타냅니다. (예: 수학식 x<sup>2</sup>, 각주 번호)
    *   **`<time>`**: 날짜, 시간 또는 날짜와 시간을 모두 나타냅니다. `datetime` 속성을 사용하여 기계가 읽을 수 있는 형식으로 시간을 제공할 수 있습니다.
    *   **`<dfn>`**: 정의하고 있는 용어의 정의 인스턴스를 나타냅니다.
    *   **`<samp>`**: 컴퓨터 프로그램의 샘플 출력(또는 스크립트의 출력)을 나타냅니다.
    *   **`<kbd>`**: 사용자 입력(일반적으로 키보드 입력)을 나타냅니다.
    *   **`<var>`**: 수학적 표현이나 프로그래밍 컨텍스트에서 변수를 나타냅니다.

    **코드 사례**:
    ```html
    <p>이 프로젝트는 <strong>반드시</strong> 내일까지 완료해야 합니다.</p>
    <p>이것은 <b>HTML</b> 학습 문서입니다.</p>
    <p>이것은 <em>정말</em> 멋진 기능입니다!</p>
    <p>그는 생각했다, <i>이게 가능할까?</i></p>
    <p>검색 결과에서 <mark>중요한</mark> 키워드를 찾았습니다.</p>
    <p><small>모든 권리 보유 &copy; 2025</small></p>
    <p>가격: <del>10,000원</del> <ins>8,000원</ins></p>
    <p>화학식: H<sub>2</sub>O, 수학식: E=mc<sup>2</sup></p>
    <p>회의는 <time datetime="2025-08-10T14:00">오늘 오후 2시</time>에 시작합니다.</p>
    <p>HTML은 <dfn>웹 페이지의 구조를 정의하는 마크업 언어</dfn>입니다.</p>
    <p>시스템 출력: <samp>Permission denied.</samp></p>
    <p>저장하려면 <kbd>Ctrl</kbd> + <kbd>S</kbd>를 누르세요.</p>
    <p>방정식 <var>x</var> + <var>y</var> = <var>z</var>를 푸시오.</p>
    ```

- **2.1.5. 인용 관련 태그**
    *   **`<blockquote>`**: 다른 출처에서 인용한 **긴 텍스트 블록**을 나타냅니다. 주로 블록 레벨 요소로 사용되며, `cite` 속성으로 인용문의 출처 URL을 명시할 수 있습니다.
    *   **`<q>`**: 짧은, **인라인(inline) 인용문**을 나타냅니다. 브라우저는 일반적으로 인용 부호(따옴표)를 자동으로 추가합니다.
    *   **`<cite>`**: 작품의 제목(책, 영화, 노래 등)이나 인용문의 출처를 나타냅니다. 시각적으로는 기울임꼴로 표시되는 경우가 많습니다.
    *   **코드 사례**:
        ```html
        <blockquote cite="https://www.w3.org/TR/html5/text-level-semantics.html#the-blockquote-element">
            <p>웹은 모든 사람을 위한 것입니다.</p>
            <footer>— <cite>팀 버너스 리</cite></footer>
        </blockquote>
        <p>팀 버너스 리는 <q cite="https://www.w3.org/TR/html5/text-level-semantics.html#the-q-element">웹은 모든 사람을 위한 것입니다.</q>라고 말했습니다.</p>
        <p>내가 가장 좋아하는 책은 <cite>어린 왕자</cite>입니다.</p>
        ```

- **2.1.6. 약어 및 연락처 정보**
    *   **`<abbr>`**: 약어(abbreviation)를 나타내며, `title` 속성으로 약어의 전체 이름을 제공하여 접근성을 높일 수 있습니다. 마우스를 올리면 전체 이름이 툴팁으로 표시됩니다.
    *   **`<address>`**: 문서나 섹션의 작성자/소유자에 대한 연락처 정보를 제공합니다. 일반적으로 `<footer>` 내에 위치합니다.
    *   **코드 사례**:
        ```html
        <p>이 문서는 <abbr title="HyperText Markup Language">HTML</abbr>에 대한 것입니다.</p>
        <footer>
            <address>
                작성자: <a href="mailto:webmaster@example.com">웹마스터</a><br>
                방문 주소: 서울시 강남구 테헤란로 123<br>
                전화: <a href="tel:+82-2-1234-5678">02-1234-5678</a>
            </address>
        </footer>
        ```

- **2.1.7. 인라인 스타일링 컨테이너: `<span>`**
    *   **역할**: 텍스트의 일부를 그룹화하여 CSS로 스타일을 적용하기 위한 **인라인(inline) 컨테이너**입니다. 그 자체로는 아무런 시각적 변화를 주지 않으며, 특정 부분에만 다른 색상, 폰트, 배경색 등을 적용하고 싶을 때 유용합니다.
    *   **시맨틱 vs 비시맨틱**: `<span>`은 의미론적인 태그가 아니므로, `<strong>`이나 `<em>`과 같이 더 적절한 시맨틱 태그가 있다면 해당 태그를 우선적으로 사용하는 것이 좋습니다. `<span>`은 오직 스타일링 목적으로만 사용될 때 적합합니다.
    *   **코드 사례**:
        ```html
        <p>이 문장에서 <span style="color: blue; font-weight: bold;">중요한 단어</span>를 강조합니다.</p>
        <h1><span style="color:red;font-weight:bold;">Learn</span> HTML</h1>
        ```

### 2.2. 목록 태그
HTML은 정보를 구조화하여 목록 형태로 보여주기 위한 세 가지 주요 태그를 제공합니다. 이 태그들을 올바르게 사용하는 것은 콘텐츠의 의미를 명확히 하고, 웹 접근성을 향상시키는 데 중요합니다.

- **2.2.1. 순서 없는 목록 (Unordered List): `<ul>`**
    *   **역할**: 항목들의 순서가 중요하지 않을 때 사용합니다. 각 목록 항목은 **`<li>`** (list item) 태그로 정의됩니다. 브라우저는 기본적으로 각 항목 앞에 점(bullet)을 표시합니다.
    *   **속성 및 스타일링**: `<ul>` 태그 자체에는 특별한 속성이 없지만, CSS의 `list-style-type` 속성을 사용하여 목록 앞의 마커(bullet) 모양을 변경할 수 있습니다.
        *   `disc` (기본값): 채워진 원
        *   `circle`: 비어있는 원
        *   `square`: 채워진 사각형
        *   `none`: 마커 없음 (가장 흔하게 사용되며, CSS로 커스텀 마커를 만들 때 유용)
    *   **중첩 목록**: 목록 안에 또 다른 목록을 중첩하여 계층적인 구조를 만들 수 있습니다. `<li>` 태그 안에 `<ul>` 또는 `<ol>` 태그를 배치합니다.
    *   **코드 사례**:
        ```html
        <h3>오늘의 할 일</h3>
        <ul>
            <li>장보기</li>
            <li>운동하기
                <ul>
                    <li>헬스장 가기</li>
                    <li>러닝 30분</li>
                </ul>
            </li>
            <li>독서하기</li>
        </ul>

        <h3>선호하는 과일 (마커 없음)</h3>
        <ul style="list-style-type: none; padding-left: 20px;">
            <li>🍎 사과</li>
            <li>🍌 바나나</li>
            <li>🍊 오렌지</li>
        </ul>
        ```

- **2.2.2. 순서 있는 목록 (Ordered List): `<ol>`**
    *   **역할**: 항목들의 순서가 중요할 때 사용합니다. 각 항목은 **`<li>`** 태그로 정의되며, 브라우저는 자동으로 번호를 매깁니다.
    *   **주요 속성**:
        *   **`type`**: 번호의 종류를 지정합니다.
            *   `1` (기본값): 숫자 (1, 2, 3...)
            *   `A`: 대문자 알파벳 (A, B, C...)
            *   `a`: 소문자 알파벳 (a, b, c...)
            *   `I`: 대문자 로마 숫자 (I, II, III...)
            *   `i`: 소문자 로마 숫자 (i, ii, iii...)
        *   **`start`**: 목록의 시작 번호를 지정합니다. (예: `start="5"`는 5부터 시작)
        *   **`reversed`**: 목록의 순서를 역순으로 표시합니다. (예: 3, 2, 1 또는 C, B, A)
    *   **중첩 목록**: `<ul>`과 마찬가지로 `<li>` 태그 안에 다른 목록을 중첩할 수 있습니다.
    *   **코드 사례**:
        ```html
        <h3>요리 레시피 단계</h3>
        <ol>
            <li>재료 준비하기</li>
            <li>오븐 예열하기</li>
            <li>반죽 섞기</li>
            <li>오븐에 넣고 굽기</li>
        </ol>

        <h3>상위 3개 영화 (역순 로마 숫자)</h3>
        <ol type="I" start="3" reversed>
            <li>영화 C</li>
            <li>영화 B</li>
            <li>영화 A</li>
        </ol>
        ```

- **2.2.3. 정의 목록 (Description List): `<dl>`**
    *   **역할**: 용어와 그에 대한 설명을 짝지어 보여줄 때 사용합니다. 사전, 용어집, 질문과 답변(FAQ) 형식의 콘텐츠에 적합합니다.
    *   **구성 요소**:
        *   **`<dl>`**: 정의 목록 전체를 감싸는 컨테이너입니다.
        *   **`<dt>` (definition term)**: 정의할 용어를 나타냅니다.
        *   **`<dd>` (definition description)**: 용어에 대한 설명을 나타냅니다.
    *   **유연한 구조**: 하나의 `<dt>`에 여러 개의 `<dd>`가 올 수 있고, 반대로 여러 개의 `<dt>`가 하나의 `<dd>`를 공유할 수도 있습니다.
    *   **코드 사례**:
        ```html
        <h3>주요 웹 용어</h3>
        <dl>
            <dt>HTML</dt>
            <dd>HyperText Markup Language의 약자로, 웹 페이지의 구조를 정의하는 마크업 언어입니다.</dd>
            <dt>CSS</dt>
            <dd>Cascading Style Sheets의 약자로, 웹 페이지의 디자인과 레이아웃을 담당합니다.</dd>
            <dt>JavaScript</dt>
            <dd>웹 페이지에 동적인 기능을 부여하는 프로그래밍 언어입니다.</dd>
            <dd>클라이언트 사이드 스크립팅에 주로 사용됩니다.</dd>
        </dl>

        <h3>자주 묻는 질문</h3>
        <dl>
            <dt>Q: 주문은 어떻게 하나요?</dt>
            <dd>A: 웹사이트에서 원하는 상품을 선택 후 장바구니에 담아 결제하시면 됩니다.</dd>
            <dt>Q: 배송은 얼마나 걸리나요?</dt>
            <dd>A: 영업일 기준 2~3일 소요됩니다.</dd>
            <dt>Q: 교환/환불 정책은 어떻게 되나요?</dt>
            <dd>A: 상품 수령 후 7일 이내에 고객센터로 문의해주세요.</dd>
        </dl>
        ```

- **2.2.4. 목록 태그의 시맨틱 사용 및 접근성**
    *   **의미론적 정확성**: 목록 태그는 단순히 시각적인 나열을 넘어 콘텐츠의 의미를 전달하는 중요한 역할을 합니다. 순서가 중요한지, 중요하지 않은지, 아니면 용어-설명 관계인지에 따라 올바른 태그를 선택해야 합니다.
    *   **웹 접근성**: 스크린 리더와 같은 보조 기술은 목록 태그를 인식하여 사용자에게 "목록 시작", "항목 1/3", "목록 끝"과 같은 정보를 제공합니다. 이는 시각 장애인 사용자가 페이지의 구조를 이해하고 탐색하는 데 필수적입니다. 잘못된 태그 사용은 이러한 정보를 왜곡하여 접근성을 저해할 수 있습니다.
    *   **CSS와의 분리**: 목록의 시각적인 표현(예: 마커 모양, 들여쓰기)은 CSS를 사용하여 제어해야 합니다. HTML 태그 자체는 콘텐츠의 구조와 의미를 정의하는 데 집중해야 합니다.

### 2.3. 링크 태그 (`<a>`)
**`<a>`** (anchor) 태그는 웹 페이지에서 다른 웹 페이지, 파일, 이메일 주소, 또는 같은 페이지 내의 특정 위치로 연결되는 **하이퍼링크**를 만듭니다. 웹의 핵심적인 요소이며, 정보의 연결성을 가능하게 합니다.

- **2.3.1. 기본 구조 및 필수 속성**
    *   **`href` (hypertext reference)**: 링크의 목적지 URL을 지정하는 **필수 속성**입니다. 이 속성값이 없으면 `<a>` 태그는 단순한 텍스트가 되며 링크로서의 기능은 하지 않습니다. 
    *   **링크 텍스트**: `<a>` 태그 사이에 위치하는 텍스트나 이미지 등이 사용자가 클릭할 수 있는 부분입니다. 이 텍스트는 링크의 목적을 명확하게 설명해야 합니다. (예: "자세히 보기" 대신 "HTML 기본 구조 자세히 보기")
    *   **코드 사례**:
        ```html
        <a href="https://www.example.com">예시 웹사이트 방문하기</a>
        ```

- **2.3.2. 다양한 유형의 링크**
    *   **외부 웹 페이지 링크**: 다른 도메인이나 웹사이트로 이동할 때 사용합니다.
        *   **`target="_blank"`**: 링크를 클릭했을 때 새 브라우저 탭이나 창에서 열리도록 합니다.
        *   **`rel="noopener noreferrer"`**: `target="_blank"`와 함께 **반드시 사용해야 하는 보안 및 성능 속성**입니다.
            *   `noopener`: 새 탭에서 열린 페이지가 `window.opener` 속성을 통해 원래 페이지의 `window` 객체에 접근하는 것을 방지하여, 피싱 공격(tabnabbing)을 막습니다.
            *   `noreferrer`: 새 탭으로 이동할 때 HTTP `Referer` 헤더를 전송하지 않도록 하여, 원래 페이지의 URL이 새 페이지로 노출되는 것을 방지합니다.
        *   **코드 사례**:
            ```html
            <a href="https://www.google.com" target="_blank" rel="noopener noreferrer">구글 검색 (새 탭)</a>
            ```
    *   **내부 문서 링크 (같은 웹사이트 내)**: 현재 웹사이트 내의 다른 페이지로 이동할 때 사용합니다. 상대 경로 또는 절대 경로를 사용할 수 있습니다.
        *   **상대 경로**: 현재 파일의 위치를 기준으로 다른 파일의 경로를 지정합니다. (예: `./about.html`, `../images/pic.jpg`)
        *   **절대 경로**: 웹사이트의 루트 디렉토리를 기준으로 경로를 지정합니다. (예: `/products/item1.html`)
        *   **코드 사례**:
            ```html
            <a href="./about.html">회사 소개 페이지로 이동</a>
            <a href="/products/item1.html">상품 상세 페이지</a>
            ```
    *   **페이지 내 특정 위치로 이동 (앵커 링크)**: 현재 페이지 내의 특정 섹션으로 스크롤하여 이동할 때 사용합니다.
        *   이동할 대상 요소에 고유한 `id` 속성을 부여하고, `href` 속성값에 `#` 기호와 함께 해당 `id` 값을 지정합니다.
        *   **코드 사례**:
            ```html
            <!-- 목차 -->
            <nav>
                <ul>
                    <li><a href="#section1">섹션 1: 개요</a></li>
                    <li><a href="#section2">섹션 2: 상세 내용</a></li>
                </ul>
            </nav>

            <!-- 페이지 내용 -->
            <h2 id="section1">섹션 1: 개요</h2>
            <p>이것은 섹션 1의 내용입니다.</p>
            <h2 id="section2">섹션 2: 상세 내용</h2>
            <p>이것은 섹션 2의 내용입니다.</p>
            ```
    *   **이메일 링크 (`mailto:`)**: 사용자의 기본 이메일 클라이언트를 열어 특정 이메일 주소로 메일을 보낼 수 있게 합니다.
        *   **추가 파라미터**: `?subject=`, `&body=`, `&cc=`, `&bcc=` 등을 사용하여 제목, 본문, 참조, 숨은 참조를 미리 채울 수 있습니다.
        *   **코드 사례**:
            ```html
            <a href="mailto:support@example.com?subject=문의사항&body=안녕하세요, 문의드립니다.">고객 지원팀에 이메일 보내기</a>
            ```
    *   **전화 걸기 링크 (`tel:`)**: 모바일 기기에서 링크 클릭 시 해당 번호로 전화를 걸 수 있게 합니다.
        *   **코드 사례**:
            ```html
            <a href="tel:+82-10-1234-5678">전화 걸기: 010-1234-5678</a>
            ```
    *   **파일 다운로드 링크**: 특정 파일을 다운로드하도록 유도합니다.
        *   **`download` 속성 (HTML5)**: `href`에 지정된 리소스를 다운로드하도록 브라우저에 지시하며, 선택적으로 다운로드될 파일의 이름을 지정할 수 있습니다.
        *   **코드 사례**:
            ```html
            <a href="/documents/report.pdf" download="연간보고서_2024.pdf">2024년 연간 보고서 다운로드</a>
            ```
    *   **이미지 링크**: `<a>` 태그 안에 `<img>` 태그를 중첩하여 이미지를 클릭 가능한 링크로 만들 수 있습니다.
        *   **코드 사례**:
            ```html
            <a href="https://www.example.com/gallery"><img src="./images/gallery_thumbnail.jpg" alt="갤러리로 이동"></a>
            ```

- **2.3.3. 링크 관련 속성 심화**
    *   **`title` 속성**: 링크에 대한 추가 정보를 제공하는 툴팁을 표시합니다. 마우스를 링크 위에 올렸을 때 나타납니다. 접근성 측면에서는 링크 텍스트 자체가 명확한 것이 더 중요합니다.
    *   **`rel` 속성 (Relationship)**: 현재 문서와 링크된 문서 간의 관계를 나타냅니다.
        *   `nofollow`: 검색 엔진 크롤러에게 이 링크를 따라가지 말고, 링크된 페이지에 페이지 랭크(PageRank)를 전달하지 말라고 지시합니다. 주로 사용자 생성 콘텐츠(댓글, 포럼 게시물)의 스팸 방지나 신뢰할 수 없는 외부 링크에 사용됩니다.
        *   `ugc` (User Generated Content): 사용자 생성 콘텐츠(댓글, 포럼 게시물 등) 내의 링크임을 나타냅니다. `nofollow`와 함께 사용될 수 있습니다.
        *   `sponsored`: 유료 링크, 광고 또는 기타 보상 계약의 결과로 생성된 링크임을 나타냅니다. `nofollow`와 함께 사용될 수 있습니다.
    *   **`type` 속성**: 링크된 리소스의 MIME 타입(예: `text/html`, `application/pdf`)을 명시합니다. 브라우저가 리소스를 가져오기 전에 해당 타입에 대한 정보를 얻을 수 있게 합니다.

- **2.3.4. 링크 사용의 모범 사례**
    *   **명확하고 간결한 링크 텍스트**: "여기 클릭"과 같은 모호한 텍스트 대신, 링크의 목적을 명확히 설명하는 텍스트를 사용합니다. (예: "제품 상세 정보 보기", "회원 가입하기"). 이는 사용자 경험과 SEO, 접근성 모두에 중요합니다.
    *   **버튼과 링크의 구분**:
        *   **링크 (`<a>`)**: 사용자를 다른 페이지나 리소스로 **이동**시키는 데 사용합니다. (Navigation)
        *   **버튼 (`<button>`)**: 특정 **액션**을 수행하는 데 사용합니다. (Action, 예: 폼 제출, 모달 열기, JavaScript 함수 실행).
        *   JavaScript를 사용하여 버튼 클릭 시 페이지를 이동시키는 것은 가능하지만, 시맨틱적으로는 `<a>` 태그를 사용하는 것이 더 적절합니다.
    *   **접근성 고려**: 스크린 리더 사용자가 링크의 목적을 쉽게 이해할 수 있도록 `aria-label`과 같은 ARIA 속성을 활용할 수 있습니다.
    *   **CSS를 통한 스타일링**: 링크의 시각적인 표현(색상, 밑줄, 호버 효과 등)은 CSS를 사용하여 제어합니다.

### 2.4. 이미지 태그 (`<img>`)
**`<img>`** 태그는 웹 페이지에 이미지를 삽입할 때 사용되는 빈 태그(empty tag)입니다. 웹 콘텐츠에서 시각적인 정보를 제공하는 데 필수적인 요소입니다.

- **2.4.1. 기본 속성 및 중요성**
    *   **`src` (source)**: 이미지 파일의 경로(URL)를 지정하는 **필수 속성**입니다. 브라우저는 이 경로를 통해 이미지를 가져와 표시합니다.
    *   **`alt` (alternative text)**: 이미지의 내용을 설명하는 **필수 속성**입니다.
        *   **접근성**: 시각 장애인 사용자를 위한 스크린 리더가 이미지를 설명하는 데 사용됩니다.
        *   **SEO**: 검색 엔진이 이미지의 내용을 이해하고 색인하는 데 도움을 줍니다.
        *   **대체 텍스트**: 네트워크 오류, 이미지 경로 문제 등으로 이미지를 표시할 수 없을 때 `alt` 속성의 텍스트가 대신 표시됩니다.
        *   **코드 사례**:
            ```html
            <img src="flower.jpg" alt="활짝 핀 붉은 장미꽃">
            ```
    *   **`width`, `height`**: 이미지의 너비와 높이를 픽셀 단위로 지정합니다.
        *   **레이아웃 안정성**: 이미지가 로드되기 전에 브라우저가 해당 공간을 미리 확보하여, 이미지가 로드되면서 페이지 레이아웃이 갑자기 변경되는 현상(Cumulative Layout Shift, CLS)을 방지합니다.
        *   **성능**: 브라우저가 이미지의 크기를 미리 알 수 있어 렌더링 효율을 높일 수 있습니다.
        *   **코드 사례**:
            ```html
            <img src="logo.png" alt="회사 로고" width="150" height="50">
            ```

- **2.4.2. 주요 이미지 형식**
    웹에서 사용되는 다양한 이미지 형식은 각각의 특징과 장단점을 가집니다.
    *   **`JPG`/`JPEG`**:
        *   **특징**: 손실 압축(lossy compression) 방식. 압축률이 높아 파일 크기가 작지만, 압축할수록 이미지 품질이 저하됩니다.
        *   **용도**: 사진과 같이 색상이 풍부하고 복잡한 이미지에 적합합니다.
    *   **`PNG`**:
        *   **특징**: 비손실 압축(lossless compression) 방식. 이미지 품질 손상 없이 압축하며, 투명 배경(알파 채널)을 지원합니다.
        *   **용도**: 로고, 아이콘, 투명 배경이 필요한 이미지, 텍스트나 선이 많은 이미지에 적합합니다.
    *   **`GIF`**:
        *   **특징**: 256색상만 지원하는 비손실 압축 방식. 여러 프레임을 포함하여 간단한 애니메이션을 만들 수 있습니다.
        *   **용도**: 간단한 애니메이션, 아이콘.
    *   **`SVG` (Scalable Vector Graphics)**:
        *   **특징**: 벡터 기반 이미지 형식. XML 형식으로 작성되며, 확대하거나 축소해도 품질이 저하되지 않습니다. 파일 크기가 작고 해상도에 독립적입니다.
        *   **용도**: 로고, 아이콘, 그래프, 일러스트레이션 등.
    *   **`WebP`**:
        *   **특징**: Google에서 개발한 차세대 이미지 형식. JPEG, PNG, GIF의 장점을 결합하여 더 높은 압축률과 품질을 제공합니다. 투명 배경 및 애니메이션 지원.
        *   **용도**: 웹 성능 최적화를 위해 다양한 유형의 이미지에 사용됩니다.
    *   **`AVIF` (AV1 Image File Format)**:
        *   **특징**: 최신 이미지 형식으로, WebP보다 더 높은 압축 효율과 품질을 제공합니다. HDR(High Dynamic Range) 및 투명도 지원.
        *   **용도**: 고품질 이미지와 최적의 성능이 요구되는 경우. 아직 모든 브라우저에서 완벽하게 지원되지는 않습니다.

- **2.4.3. 반응형 이미지 구현**
    다양한 화면 크기와 해상도에 최적화된 이미지를 제공하기 위해 반응형 이미지 기법을 사용합니다.
    *   **`srcset` 및 `sizes` 속성**:
        *   **`srcset`**: 브라우저에 여러 해상도의 이미지 소스를 제공합니다. 브라우저는 사용자의 화면 해상도(픽셀 밀도)와 뷰포트 크기에 따라 가장 적합한 이미지를 선택하여 로드합니다.
        *   **`sizes`**: 이미지가 뷰포트의 특정 너비에서 차지할 공간을 브라우저에 알려줍니다. `srcset`과 함께 사용하여 브라우저가 올바른 이미지를 선택하도록 돕습니다.
        *   **코드 사례**:
            ```html
            <img
                src="image-small.jpg"
                srcset="image-small.jpg 480w, image-medium.jpg 800w, image-large.jpg 1200w"
                sizes="(max-width: 600px) 480px, (max-width: 1000px) 800px, 1200px"
                alt="다양한 크기에 최적화된 이미지"
            >
            ```
    *   **`<picture>` 요소**:
        *   **아트 디렉션 (Art Direction)**: 뷰포트 크기나 장치 특성(예: 가로/세로 비율)에 따라 이미지를 자르거나 다른 버전의 이미지를 제공할 때 사용합니다.
        *   **형식 지원**: 브라우저가 특정 이미지 형식을 지원하는지 여부에 따라 다른 형식의 이미지를 제공할 수 있습니다. (예: WebP를 지원하면 WebP를, 아니면 JPEG를 로드).
        *   **`<source>` 태그**: `media` 속성으로 미디어 쿼리를, `type` 속성으로 이미지 형식을 지정하여 조건에 맞는 이미지를 제공합니다.
        *   **코드 사례**:
            ```html
            <picture>
              <source srcset="images/hero-large.webp" type="image/webp" media="(min-width: 1200px)">
              <source srcset="images/hero-medium.webp" type="image/webp" media="(min-width: 768px)">
              <source srcset="images/hero-small.webp" type="image/webp">
              <img src="images/hero-fallback.jpg" alt="반응형 히어로 이미지">
            </picture>
            ```

- **2.4.4. 이미지 성능 최적화**
    *   **지연 로딩 (Lazy Loading)**: `loading="lazy"` 속성을 사용하여 뷰포트 밖에 있는 이미지를 사용자가 스크롤하여 볼 때까지 로드를 지연시킵니다. 이는 초기 페이지 로드 시간을 단축하고 대역폭을 절약하는 데 도움이 됩니다.
        *   **코드 사례**:
            ```html
            <img src="image.jpg" alt="지연 로딩 이미지" loading="lazy">
            ```
    *   **이미지 압축 및 크기 조정**: 이미지를 웹에 업로드하기 전에 적절한 크기로 조정하고 압축하여 파일 크기를 최소화합니다.
    *   **CDN (Content Delivery Network) 사용**: 이미지를 CDN을 통해 제공하여 사용자에게 가장 가까운 서버에서 이미지를 전송함으로써 로딩 속도를 향상시킵니다.
    *   **`decoding="async"`**: 이미지 디코딩을 비동기적으로 처리하여 메인 스레드를 차단하지 않도록 합니다. 페이지의 다른 콘텐츠 렌더링에 영향을 주지 않아야 하는 비필수 이미지에 유용합니다.

- **2.4.5. `<img>` 태그와 CSS `background-image`의 차이**
    두 가지 모두 이미지를 표시하는 데 사용되지만, 목적과 시맨틱 의미가 다릅니다.
    *   **`<img>` 태그**:
        *   **용도**: 이미지가 **콘텐츠의 일부**이며, 문서의 의미론적 구조에 중요한 역할을 할 때 사용합니다. (예: 상품 이미지, 기사 내 삽화, 사용자 프로필 사진).
        *   **SEO 및 접근성**: `alt` 속성을 통해 이미지의 내용을 검색 엔진과 스크린 리더에 전달할 수 있습니다.
    *   **CSS `background-image`**:
        *   **용도**: 이미지가 순전히 **장식적인 목적**으로 사용될 때 적합합니다. (예: 페이지 배경, UI 요소의 아이콘, 버튼의 배경).
        *   **SEO 및 접근성**: `background-image`는 콘텐츠의 일부로 간주되지 않으므로, 검색 엔진이 인덱싱하지 않으며 스크린 리더도 읽지 않습니다. 중요한 정보는 `background-image`로 제공해서는 안 됩니다.

### 2.5. 테이블 태그
**`<table>`** 태그는 데이터를 **행(row)과 열(column)의 표 형식**으로 나타낼 때 사용합니다. 과거에는 웹 페이지의 레이아웃을 잡기 위해 사용되기도 했지만, 이는 시맨틱적으로 올바르지 않으며 웹 표준에 따라 **오직 표 형식의 데이터 표시에만 사용**하는 것이 강력히 권장됩니다.

- **2.5.1. 테이블의 기본 구조**
    *   **`<table>`**: 테이블 전체를 감싸는 최상위 컨테이너입니다.
    *   **`<tr>` (table row)**: 테이블의 **행(row)**을 정의합니다. 각 `<tr>` 태그는 테이블의 한 줄을 나타냅니다.
    *   **`<th>` (table header)**: 테이블의 **제목(헤더) 셀**을 정의합니다. 기본적으로 텍스트가 굵게 표시되고 가운데 정렬됩니다. `scope` 속성을 사용하여 해당 헤더가 어떤 셀들의 제목인지 명시할 수 있습니다.
    *   **`<td>` (table data)**: 테이블의 **데이터 셀(칸)**을 정의합니다. 실제 데이터가 들어가는 부분입니다.
    *   **코드 사례**:
        ```html
        <table border="1">
            <tr>
                <th>이름</th>
                <th>나이</th>
                <th>직업</th>
            </tr>
            <tr>
                <td>김철수</td>
                <td>30</td>
                <td>개발자</td>
            </tr>
            <tr>
                <td>이영희</td>
                <td>25</td>
                <td>디자이너</td>
            </tr>
        </table>
        ```

- **2.5.2. 테이블의 시맨틱 그룹화 및 접근성**
    테이블의 구조를 명확히 하고 접근성을 높이기 위해 다음과 같은 태그들을 사용합니다.
    *   **`<caption>`**: 테이블의 **제목이나 설명**을 제공합니다. `<table>` 태그 바로 다음에 위치해야 하며, 스크린 리더 사용자에게 테이블의 내용을 빠르게 파악할 수 있도록 돕습니다.
    *   **`<thead>` (table head)**: 테이블의 **머리글(헤더) 행 그룹**을 정의합니다. 일반적으로 `<th>` 태그를 포함합니다.
    *   **`<tbody>` (table body)**: 테이블의 **본문(데이터) 행 그룹**을 정의합니다. 일반적으로 `<td>` 태그를 포함합니다.
    *   **`<tfoot>` (table foot)**: 테이블의 **바닥글(푸터) 행 그룹**을 정의합니다. 합계나 요약 정보 등을 포함할 수 있습니다.
    *   **`scope` 속성 (`<th>`에 사용)**:
        *   `scope="col"`: 해당 `<th>`가 열(column)의 제목임을 나타냅니다.
        *   `scope="row"`: 해당 `<th>`가 행(row)의 제목임을 나타냅니다.
        *   이 속성은 스크린 리더가 테이블 데이터를 읽을 때 각 셀이 어떤 헤더에 속하는지 정확히 파악하도록 도와 접근성을 크게 향상시킵니다.
    *   **코드 사례 (시맨틱 그룹화)**:
        ```html
        <table border="1">
            <caption>2024년 분기별 매출 현황</caption>
            <thead>
                <tr>
                    <th scope="col">분기</th>
                    <th scope="col">제품 A</th>
                    <th scope="col">제품 B</th>
                    <th scope="col">총 매출</th>
                </tr>
            </thead>
            <tbody>
                <tr>
                    <th scope="row">1분기</th>
                    <td>1000</td>
                    <td>1500</td>
                    <td>2500</td>
                </tr>
                <tr>
                    <th scope="row">2분기</th>
                    <td>1200</td>
                    <td>1800</td>
                    <td>3000</td>
                </tr>
            </tbody>
            <tfoot>
                <tr>
                    <th scope="row" colspan="3">상반기 합계</th>
                    <td>5500</td>
                </tr>
            </tfoot>
        </table>
        ```

- **2.5.3. 셀 병합: `rowspan` 및 `colspan`**
    *   **`rowspan`**: 현재 셀이 병합할 **행의 개수**를 지정합니다. (세로 병합)
    *   **`colspan`**: 현재 셀이 병합할 **열의 개수**를 지정합니다. (가로 병합)
    *   **코드 사례**:
        ```html
        <table border="1">
            <caption>학생 성적표</caption>
            <thead>
                <tr>
                    <th rowspan="2">이름</th>
                    <th colspan="2">점수</th>
                </tr>
                <tr>
                    <th>수학</th>
                    <th>영어</th>
                </tr>
            </thead>
            <tbody>
                <tr>
                    <td>김민수</td>
                    <td>90</td>
                    <td>85</td>
                </tr>
                <tr>
                    <td>박지민</td>
                    <td>75</td>
                    <td>92</td>
                </tr>
            </tbody>
        </table>
        ```

- **2.5.4. 열 그룹화: `<colgroup>` 및 `<col>`**
    *   **`<colgroup>`**: 하나 이상의 `<col>` 태그를 포함하는 컨테이너로, 테이블의 열들을 그룹화합니다.
    *   **`<col>`**: `<colgroup>` 내에서 특정 열 또는 여러 열에 대한 속성을 정의합니다. 주로 스타일링(예: 배경색, 너비)을 적용하거나 시맨틱적인 의미를 부여할 때 사용됩니다. `span` 속성을 사용하여 여러 열에 동시에 적용할 수 있습니다.
    *   **코드 사례**:
        ```html
        <table border="1">
            <caption>제품 재고 현황</caption>
            <colgroup>
                <col style="background-color: #f2f2f2;"> <!-- 첫 번째 열 -->
                <col span="2" style="width: 150px;"> <!-- 두 번째, 세 번째 열 -->
                <col style="background-color: #e6f7ff;"> <!-- 네 번째 열 -->
            </colgroup>
            <thead>
                <tr>
                    <th>제품명</th>
                    <th>재고</th>
                    <th>단위</th>
                    <th>상태</th>
                </tr>
            </thead>
            <tbody>
                <tr>
                    <td>노트북</td>
                    <td>50</td>
                    <td>개</td>
                    <td>정상</td>
                </tr>
                <tr>
                    <td>마우스</td>
                    <td>120</td>
                    <td>개</td>
                    <td>정상</td>
                </tr>
            </tbody>
        </table>
        ```

- **2.5.5. 테이블 스타일링 및 모범 사례**
    *   **CSS를 통한 스타일링**: 테이블의 시각적인 표현(테두리, 셀 간격, 배경색, 텍스트 정렬, 호버 효과 등)은 CSS를 사용하여 제어하는 것이 좋습니다. `border="1"`과 같은 HTML 속성은 구식이며 CSS로 대체되어야 합니다.
    *   **반응형 테이블**: 작은 화면에서 테이블이 너무 넓어지지 않도록 `overflow-x: auto;`를 사용하여 가로 스크롤을 허용하거나, CSS 미디어 쿼리를 사용하여 테이블 레이아웃을 변경하는 등의 반응형 디자인 기법을 적용해야 합니다.
    *   **복잡한 테이블의 접근성**: 매우 복잡한 테이블의 경우, `id`와 `headers` 속성을 사용하여 데이터 셀과 헤더 셀 간의 관계를 명시적으로 연결할 수 있습니다. 이는 스크린 리더 사용자가 복잡한 테이블을 이해하는 데 큰 도움이 됩니다.
    *   **데이터 정렬**: 숫자 데이터는 오른쪽 정렬, 텍스트 데이터는 왼쪽 정렬하는 것이 일반적입니다.
    *   **테이블 사용의 원칙**: 테이블은 오직 표 형식의 데이터를 표시하기 위한 것이며, 웹 페이지의 레이아웃을 구성하는 용도로 사용해서는 안 됩니다. 레이아웃은 CSS Flexbox나 Grid와 같은 현대적인 CSS 기술을 사용해야 합니다.

### 2.6. 폼 태그 (`<form>`)
**`<form>`** 태그는 사용자로부터 텍스트 입력, 항목 선택, 파일 첨부 등의 입력을 받아 서버로 전송하기 위한 영역을 정의합니다. 웹 애플리케이션에서 사용자 상호작용의 핵심적인 부분입니다.

- **2.6.1. `<form>` 태그의 주요 속성**
    *   **`action`**: 폼 데이터가 전송될 서버 측 스크립트의 URL을 지정합니다. 이 URL은 폼이 제출될 때 HTTP 요청의 대상이 됩니다.
    *   **`method`**: 폼 데이터가 서버로 전송될 때 사용될 HTTP 메서드를 지정합니다.
        *   **`GET`**:
            *   데이터가 URL의 쿼리 문자열(`?key1=value1&key2=value2`)에 포함되어 전송됩니다.
            *   URL에 노출되므로 민감한 정보(비밀번호 등) 전송에는 부적합합니다.
            *   전송할 수 있는 데이터 양에 제한이 있습니다 (브라우저 및 서버에 따라 다름).
            *   북마크 가능하며, 뒤로 가기 버튼을 눌러도 문제가 없습니다 (멱등성).
            *   주로 검색 폼이나 필터링 옵션 등 데이터를 조회하는 용도로 사용됩니다.
        *   **`POST`**:
            *   데이터가 HTTP 요청 본문(body)에 포함되어 전송됩니다. URL에 노출되지 않습니다.
            *   대용량 데이터(파일 업로드 등) 전송에 적합하며, 데이터 양에 제한이 거의 없습니다.
            *   민감한 정보 전송에 적합합니다.
            *   멱등성이 보장되지 않으므로, 동일한 요청을 여러 번 보내면 다른 결과가 발생할 수 있습니다 (예: 주문 중복).
            *   주로 데이터 생성, 수정, 삭제 등 서버의 상태를 변경하는 용도로 사용됩니다.
    *   **`enctype`**: `method`가 `POST`일 때, 폼 데이터가 서버로 전송될 때의 인코딩 유형을 지정합니다.
        *   `application/x-www-form-urlencoded` (기본값): 모든 문자를 URL 인코딩합니다.
        *   **`multipart/form-data`**: 파일 업로드 시 **반드시 사용**해야 합니다. 텍스트 데이터와 바이너리 데이터를 함께 전송할 수 있도록 합니다.
        *   `text/plain`: 모든 공백 문자를 `+`로 변환하고, 특수 문자는 인코딩하지 않습니다. 디버깅 목적으로만 사용됩니다.
    *   **`autocomplete`**: 폼 또는 개별 입력 필드의 자동 완성 기능을 제어합니다 (`on` 또는 `off`).
    *   **`novalidate`**: 폼 제출 시 브라우저의 기본 HTML5 유효성 검사를 비활성화합니다. (서버 측 유효성 검사를 직접 구현할 때 유용)

- **2.6.2. 폼 컨트롤 요소**
    사용자로부터 다양한 형태의 입력을 받기 위한 요소들입니다.
    *   **`<input>`**: `type` 속성에 따라 다양한 형태의 입력 필드를 만듭니다.
        *   **텍스트 입력**: `text`, `password`, `email`, `number`, `tel`, `url`, `search`
        *   **선택 입력**: `checkbox`, `radio`
        *   **날짜/시간**: `date`, `time`, `datetime-local`, `month`, `week`
        *   **범위/색상**: `range`, `color`
        *   **파일**: `file` (파일 업로드)
        *   **버튼**: `submit`, `reset`, `button`
        *   **숨겨진 값**: `hidden` (사용자에게 보이지 않지만 서버로 전송될 값)
        *   **공통 속성**:
            *   **`name`**: 각 입력 요소의 이름을 지정하며, 서버로 데이터를 전송할 때 `key=value` 쌍에서 `key` 역할을 합니다. **필수적입니다.**
            *   **`value`**: 해당 입력 요소의 초기값 또는 선택되었을 때 서버로 전송될 값입니다.
            *   **`placeholder`**: 입력 필드에 사용자가 어떤 값을 입력해야 하는지 알려주는 힌트 텍스트를 표시합니다.
            *   **`readonly`**: 값을 수정할 수 없지만, 폼 데이터는 서버로 전송됩니다.
            *   **`disabled`**: 요소를 완전히 비활성화하여 사용 및 클릭이 불가능하게 만들고, 데이터도 서버로 전송되지 않습니다.
            *   **`autofocus`**: 페이지 로드 시 자동으로 포커스를 받을 입력 필드를 지정합니다.
    *   **`<textarea>`**: 여러 줄의 텍스트를 입력받는 필드입니다. `rows` (보이는 행 수)와 `cols` (보이는 열 수) 속성으로 크기를 조절할 수 있습니다.
    *   **`<select>` 및 `<option>`**: 드롭다운 목록을 만듭니다.
        *   **`<option>`**: 드롭다운 목록의 각 항목을 정의합니다. `value` 속성은 서버로 전송될 값을, 태그 사이의 텍스트는 사용자에게 표시될 값을 나타냅니다.
        *   **`multiple` 속성**: `<select>`에 추가하면 다중 선택이 가능해집니다.
        *   **`size` 속성**: `<select>`에 추가하면 한 번에 표시될 항목의 수를 지정합니다.
    *   **`<optgroup>`**: `<select>` 태그 내에서 관련 `<option>`들을 그룹화할 때 사용합니다. `label` 속성으로 그룹의 제목을 지정합니다.
    *   **`<datalist>`**: `<input>` 태그와 함께 사용하여 사용자에게 입력 제안 목록을 제공합니다. 드롭다운 목록처럼 보이지만, 사용자가 직접 입력할 수도 있습니다. `<input>`의 `list` 속성값과 `<datalist>`의 `id` 속성값을 일치시켜 연결합니다.
    *   **`<button>`**: 클릭 가능한 버튼을 만듭니다. `type` 속성에 따라 동작이 달라집니다.
        *   `type="submit"` (기본값): 폼을 제출합니다.
        *   `type="reset"`: 폼의 모든 입력 필드를 초기화합니다.
        *   `type="button"`: 아무런 기본 동작이 없으며, JavaScript를 통해 동작을 정의합니다.
        *   `<input type="submit">`보다 유연하며, `<img>`나 `<strong>` 같은 다른 HTML 태그를 내부에 포함하여 더 복잡한 스타일링이 가능합니다.

- **2.6.3. 폼 요소 구조화 및 접근성**
    *   **`<label>` 태그**: 입력 필드에 대한 설명을 나타냅니다. **`for`** 속성값을 입력 요소의 **`id`** 속성값과 일치시키면, 라벨을 클릭했을 때 해당 입력 요소에 포커스가 이동하여 사용자 편의성과 접근성이 향상됩니다. 스크린 리더 사용자에게도 어떤 라벨이 어떤 입력 필드에 대한 것인지 명확히 알려줍니다.
    *   **`<fieldset>`과 `<legend>`**: 폼 안에서 관련된 여러 입력 요소들을 그룹화할 때 사용합니다.
        *   **`<fieldset>`**: 그룹을 감싸는 테두리를 생성합니다.
        *   **`<legend>`**: `<fieldset>`의 제목을 제공합니다.
        *   이는 시각적으로 그룹을 구분할 뿐만 아니라, 스크린 리더 사용자에게도 그룹의 목적을 알려주어 복잡한 폼을 이해하는 데 도움을 줍니다.

- **2.6.4. 클라이언트 측 유효성 검사 (Client-Side Validation)**
    HTML5는 폼 입력에 대한 클라이언트 측 유효성 검사를 위한 다양한 속성을 제공합니다. 이는 사용자 경험을 향상시키고 불필요한 서버 요청을 줄이지만, **보안과 데이터 무결성을 위해 서버 측 유효성 검사는 필수적**입니다. 클라이언트 측 유효성 검사는 사용자가 쉽게 우회할 수 있으므로, 서버 측 검사를 절대 대체할 수 없습니다.
    *   **`required`**: 폼 전송 시 해당 필드가 반드시 채워져 있어야 함을 나타냅니다.
    *   **`min`, `max`**: `number` 또는 `range` 타입 입력 필드의 최소/최대 허용 값을 지정합니다. `date`, `time` 등에도 사용 가능합니다.
    *   **`minlength`, `maxlength`**: 텍스트 입력 필드의 최소/최대 문자 길이를 지정합니다.
    *   **`pattern`**: 정규 표현식(Regular Expression)을 사용하여 입력 값의 형식을 검사합니다. `title` 속성과 함께 사용하여 사용자에게 유효성 검사 규칙에 대한 힌트를 제공하는 것이 좋습니다.
    *   **`type="email"`, `type="url"` 등**: 브라우저가 해당 타입에 맞는 기본적인 형식 유효성 검사를 수행합니다.
    *   **코드 사례**:
        ```html
        <input type="email" required placeholder="이메일 주소를 입력하세요">
        <input type="number" min="1" max="100" value="50">
        <input type="password" minlength="8" pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}" title="최소 8자, 대문자, 소문자, 숫자 포함">
        ```

- **2.6.5. 풀스택 관점: 폼 데이터 처리 및 보안**
    *   **서버 측 처리**: 폼 데이터는 `action` 속성에 지정된 URL로 전송되며, 백엔드 서버는 이 데이터를 파싱하여 비즈니스 로직을 수행합니다. 모든 사용자 입력은 잠재적으로 악의적일 수 있으므로, **서버 측에서 반드시 유효성 검사, 살균(sanitization), 그리고 적절한 에러 처리를 수행**해야 합니다.
    *   **AJAX 폼 제출**: 현대 웹 애플리케이션에서는 페이지 새로고침 없이 JavaScript(Fetch API, Axios 등)를 사용하여 폼 데이터를 비동기적으로 서버에 제출하고 응답을 처리하는 방식이 흔히 사용됩니다. 이는 사용자 경험을 향상시키고 SPA(Single Page Application) 개발에 필수적입니다.
        *   **`FormData` 객체**: JavaScript의 `FormData` 객체를 사용하면 `<form>` 태그의 데이터를 쉽게 캡처하여 `Fetch API`나 `XMLHttpRequest`를 통해 서버로 전송할 수 있습니다. 파일 업로드와 같은 `multipart/form-data` 형식의 데이터 전송에 특히 유용합니다.
    *   **보안 고려사항**:
        *   **입력 유효성 검사 (Input Validation)**: 클라이언트와 서버 양쪽에서 모든 사용자 입력을 검증하여 예상치 못한 데이터나 악의적인 코드가 시스템에 주입되는 것을 방지합니다.
        *   **입력 살균 (Input Sanitization)**: 사용자 입력에서 잠재적으로 위험한 문자(예: HTML 태그, 스크립트)를 제거하거나 이스케이프 처리하여 XSS(Cross-Site Scripting) 공격을 방지합니다.
        *   **CSRF (Cross-Site Request Forgery) 방지**: 중요한 작업을 수행하는 폼에는 CSRF 토큰을 사용하여 공격자가 사용자의 세션을 가로채어 원치 않는 요청을 보내는 것을 방지합니다.
        *   **SQL Injection 방지**: 데이터베이스 쿼리를 생성할 때 사용자 입력을 직접 사용하지 않고, Prepared Statement나 ORM(Object-Relational Mapping)을 사용하여 SQL Injection 공격을 방지합니다.

### 2.7. 그룹화 및 레이아웃 (`<div>`, `<span>`, 시맨틱 태그)
웹 페이지의 콘텐츠를 구조적으로 묶고, 전체적인 레이아웃을 구성하는 데 사용되는 HTML 태그들입니다. HTML5에서는 단순히 시각적인 그룹화를 넘어, 콘텐츠의 의미를 명확히 하는 **시맨틱(Semantic) 태그**의 사용을 강력히 권장합니다.

- **2.7.1. 비시맨틱 그룹화 태그: `<div>` 및 `<span>`**
    이 태그들은 그 자체로는 특별한 의미를 가지지 않으며, 주로 CSS를 통해 스타일을 적용하거나 JavaScript로 동적인 기능을 부여하기 위한 컨테이너 역할을 합니다.
    *   **`<div>` (Division)**:
        *   **역할**: 콘텐츠를 그룹화하는 데 사용되는 대표적인 **블록 레벨(block-level) 요소**입니다. `<div>`는 항상 새로운 줄에서 시작하며, 가능한 모든 너비를 차지합니다.
        *   **주요 용도**: 주로 CSS와 함께 페이지의 특정 구역(예: 섹션, 컬럼)이나 레이아웃을 정의할 때 사용됩니다. 다른 시맨틱 태그로 표현하기 어려운, 순수하게 스타일링이나 스크립팅을 위한 컨테이너가 필요할 때 사용합니다.
        *   **코드 사례**:
            ```html
            <div id="wrapper">
                <div class="header-container">
                    <!-- 헤더 내용 -->
                </div>
                <div class="content-area">
                    <!-- 주요 내용 -->
                </div>
            </div>
            ```
    *   **`<span>`**:
        *   **역할**: `<div>`와 비슷하게 콘텐츠를 그룹화하지만, **인라인 레벨(inline-level) 요소**라는 점이 다릅니다. `<span>`은 새로운 줄을 만들지 않고, 필요한 만큼의 공간만 차지합니다.
        *   **주요 용도**: 주로 문장 안의 특정 단어나 구절에 별도의 스타일(예: 색상, 폰트 크기)을 적용하거나, JavaScript로 특정 텍스트 조각을 조작할 때 사용됩니다.
        *   **코드 사례**:
            ```html
            <p>이 문장에서 <span style="color: blue; font-weight: bold;">중요한 단어</span>를 강조합니다.</p>
            ```

- **2.7.2. HTML5 시맨틱 레이아웃 태그**
    HTML5에서 도입된 시맨틱 태그들은 웹 페이지의 구조를 의미론적으로 명확하게 정의하여, 개발자, 브라우저, 검색 엔진, 그리고 보조 기술(스크린 리더 등)이 콘텐츠의 목적과 관계를 더 잘 이해할 수 있도록 돕습니다. 이는 웹 표준 준수, SEO, 접근성, 그리고 코드의 가독성 및 유지보수성 향상에 크게 기여합니다.

    *   **`<header>`**:
        *   **역할**: 문서나 섹션의 **머리말**을 나타냅니다. 일반적으로 제목(`<h1>`~`<h6>`), 로고, 내비게이션 링크, 검색 폼 등을 포함합니다.
        *   **주의**: `<body>` 안에 여러 개의 `<header>`가 있을 수 있습니다 (예: `<article>` 내의 `<header>`).
    *   **`<nav>`**:
        *   **역할**: 문서의 주요 **내비게이션 링크**들의 집합을 나타냅니다. 웹사이트의 주요 메뉴, 목차, 빵 부스러기(breadcrumbs) 등에 사용됩니다.
        *   **주의**: 모든 링크 그룹이 `<nav>`가 될 필요는 없습니다. 주요 내비게이션에만 사용합니다.
    *   **`<main>`**:
        *   **역할**: 문서의 **핵심적인 주요 콘텐츠**를 나타냅니다. 이 콘텐츠는 문서의 중심 주제 또는 애플리케이션의 중심 기능을 직접적으로 다룹니다.
        *   **주의**: 한 문서에 오직 하나의 `<main>` 요소만 존재해야 하며, `<body>`의 직접적인 자식이어야 합니다.
    *   **`<article>`**:
        *   **역할**: 독립적으로 배포되거나 재사용될 수 있는 **독립적인 콘텐츠**를 나타냅니다. (예: 블로그 게시물, 뉴스 기사, 사용자 댓글, 위젯).
        *   **특징**: `<article>` 내의 콘텐츠는 `<article>` 외부의 콘텐츠와 독립적으로 의미를 가집니다. 자체적인 `<header>`, `<footer>`를 가질 수 있습니다.
    *   **`<section>`**:
        *   **역할**: 문서 내에서 **관련된 콘텐츠들의 구역**을 나타냅니다. 일반적으로 제목(`<h1>`~`<h6>`)을 포함하며, 콘텐츠의 논리적인 그룹화를 위해 사용됩니다.
        *   **`<div>`와의 차이**: `<div>`는 의미가 없는 일반적인 컨테이너인 반면, `<section>`은 의미론적인 그룹화를 나타냅니다. `<section>`을 사용할지 `<div>`를 사용할지 고민될 때는, 해당 콘텐츠 그룹에 제목을 붙일 수 있는지 여부를 기준으로 판단할 수 있습니다.
        *   **`<article>`과의 관계**: `<article>` 안에 여러 `<section>`이 있을 수 있고, `<section>` 안에 여러 `<article>`이 있을 수도 있습니다.
    *   **`<aside>`**:
        *   **역할**: 주요 콘텐츠와 간접적으로 관련된 부분을 나타냅니다. (예: 사이드바, 광고 블록, 관련 링크, 인용문).
        *   **특징**: `<aside>` 내부의 콘텐츠는 주요 콘텐츠와 분리되어도 문서의 의미에 큰 영향을 주지 않습니다.
    *   **`<footer>`**:
        *   **역할**: 문서나 섹션의 **꼬리말**을 나타냅니다. 일반적으로 저작권 정보, 연락처 정보, 관련 문서 링크, 작성자 정보 등을 포함합니다.
        *   **주의**: `<body>` 안에 여러 개의 `<footer>`가 있을 수 있습니다 (예: `<article>` 내의 `<footer>`).

- **2.7.3. 시맨틱 레이아웃 구조의 예시**

- **2.7.4. `div`와 시맨틱 태그의 선택 기준**
    *   **시맨틱 태그 우선**: 콘텐츠에 의미론적으로 적합한 HTML5 시맨틱 태그가 있다면, 항상 `<div>` 대신 해당 태그를 사용해야 합니다. 이는 웹의 의미론적 구조를 강화하고, 모든 사용자에게 더 나은 경험을 제공합니다.
    *   **`div`의 사용**: `<div>`는 오직 콘텐츠를 그룹화하는 데 다른 시맨틱 태그가 적합하지 않고, 순수하게 스타일링(CSS)이나 스크립팅(JavaScript)을 위한 컨테이너가 필요할 때만 사용해야 합니다. 예를 들어, 특정 CSS 그리드 레이아웃을 위한 래퍼(wrapper)나, JavaScript로 동적으로 콘텐츠를 삽입할 빈 공간을 만들 때 유용합니다.
    *   **ARIA 역할 (Role)**: HTML5 시맨틱 태그만으로 충분하지 않거나, 커스텀 컴포넌트에 의미를 부여해야 할 경우, ARIA(Accessible Rich Internet Applications) `role` 속성을 사용하여 추가적인 의미를 부여할 수 있습니다. (예: `<div role="search">` 또는 `<div role="banner">`).

이처럼 HTML5 시맨틱 태그를 올바르게 활용하는 것은 현대 웹 개발에서 필수적인 요소이며, 단순히 시각적인 레이아웃을 넘어 웹 콘텐츠의 본질적인 의미를 전달하는 데 중요한 역할을 합니다.

### 2.8. 커스텀 데이터 속성 (`data-*` attributes)

`data-*` 속성은 HTML 요소에 사용자 정의 데이터를 저장하기 위한 표준적인 방법을 제공합니다. 이 속성들은 페이지에 시각적인 영향을 주지 않으면서, JavaScript에서 쉽게 접근하고 조작할 수 있어 동적인 웹 애플리케이션 개발에서 매우 유용하게 활용됩니다.

-   **2.8.1. 개요 및 도입 배경 (Overview and Background)**
    *   `data-*` 속성은 표준 HTML 요소에 **사용자 정의 데이터**를 저장하기 위한 공식적인 방법입니다.
    *   과거에는 `class` 속성을 남용하거나, 비표준 속성(예: `my-custom-attr="value"`)을 추가하여 스크립트에 필요한 데이터를 저장하곤 했습니다. 이는 HTML 유효성 문제를 일으키고 코드의 의도를 불분명하게 만들었습니다. `data-*` 속성은 이러한 문제들을 해결하고, 데이터를 HTML과 JavaScript 간에 안전하고 의미론적으로 전달하는 통로를 제공합니다.

-   **2.8.2. 문법 및 명명 규칙 (Syntax and Naming Rules)**
    *   **`data-` 접두사**: 속성 이름은 반드시 `data-`로 시작해야 합니다.
    *   **이름 형식**: 접두사 뒤에는 최소 하나 이상의 문자가 와야 합니다. 속성 이름에는 ASCII 대문자를 사용할 수 없습니다 (소문자, 숫자, 하이픈, 점, 콜론, 언더스코어만 사용 가능).
    *   **JavaScript 변환**: JavaScript의 `dataset` 객체에서 접근할 때, 하이픈(`-`)으로 연결된 이름은 카멜 케이스(camelCase)로 자동 변환됩니다.
        *   `data-user-id` → `dataset.userId`
        *   `data-item-name` → `dataset.itemName`

-   **2.8.3. JavaScript와의 상호작용 (Interaction with JavaScript)**
    *   `data-*` 속성의 가장 주된 용도는 JavaScript와의 연동입니다.
    *   **데이터 읽기 (Reading Data)**:
        *   **`dataset` 속성 (권장)**: 가장 간편하고 현대적인 방법입니다.
            ```javascript
            const element = document.getElementById('user-profile');
            const userId = element.dataset.userId; // '123'
            const displayMode = element.dataset.displayMode; // 'card'
            ```
        *   **`getAttribute()` 메서드**: 전통적인 방법으로, 전체 속성 이름을 사용해야 합니다.
            ```javascript
            const userId = element.getAttribute('data-user-id'); // '123'
            ```
    *   **데이터 쓰기 및 수정 (Writing & Modifying Data)**:
        *   `dataset` 속성을 통해 값을 할당하면 HTML 요소의 해당 `data-*` 속성이 동적으로 추가되거나 변경됩니다.
            ```javascript
            element.dataset.userId = '456'; // <div id="user-profile" data-user-id="456" ...>
            element.dataset.lastUpdated = new Date().toISOString(); // 새로운 속성 추가
            ```
    *   **데이터 삭제 (Deleting Data)**:
        *   `delete` 연산자를 사용합니다.
            ```javascript
            delete element.dataset.displayMode; // data-display-mode 속성 제거
            ```

-   **2.8.4. CSS와의 연동 (Interaction with CSS)**
    *   `data-*` 속성은 CSS 선택자(selector)로 활용하여 요소의 상태에 따라 스타일을 동적으로 변경하는 데 매우 유용합니다.
    *   **속성 선택자 (Attribute Selectors)**:
        ```css
        /* data-status가 "active"인 요소에 스타일 적용 */
        [data-status="active"] {
          border-color: green;
          font-weight: bold;
        }

        /* data-user-role이 "admin"인 요소만 보이게 하기 */
        .user-list .user {
          display: none;
        }
        .user-list .user[data-user-role="admin"] {
          display: block;
        }
        ```
    *   **`attr()` 함수로 데이터 표시**: CSS의 `attr()` 함수를 사용하면 `data-*` 속성 값을 콘텐츠로 가져와 표시할 수 있습니다. 주로 툴팁(tooltip) 구현에 사용됩니다.
        ```html
        <span data-tooltip="이것은 중요한 정보입니다.">도움말</span>
        ```
        ```css
        span[data-tooltip] {
          position: relative;
        }
        /* 가상 요소 ::after를 사용해 툴팁 생성 */
        span[data-tooltip]::after {
          content: attr(data-tooltip); /* data-tooltip 속성 값을 가져옴 */
          position: absolute;
          bottom: 100%;
          left: 50%;
          transform: translateX(-50%);
          background-color: #333;
          color: white;
          padding: 5px 10px;
          border-radius: 4px;
          white-space: nowrap;
          display: none; /* 평소에는 숨김 */
        }
        /* 마우스를 올리면 툴팁 표시 */
        span[data-tooltip]:hover::after {
          display: block;
        }
        ```

-   **2.8.5. 주요 활용 사례 (Practical Use Cases)**
    *   **UI 상태 제어**: 탭, 아코디언, 모달 창 등의 상태(`open`, `closed`, `active`)를 저장하고 제어합니다.
        ```html
        <div class="accordion">
            <button class="accordion-header" data-target="#panel1">섹션 1</button>
            <div id="panel1" class="accordion-panel" data-state="closed">...</div>
            <button class="accordion-header" data-target="#panel2">섹션 2</button>
            <div id="panel2" class="accordion-panel" data-state="closed">...</div>
        </div>
        ```
    *   **JavaScript 라이브러리 설정**: 간단한 설정을 JavaScript 코드 대신 HTML에 직접 명시하여 라이브러리를 초기화합니다.
        ```html
        <div class="image-slider" data-speed="300" data-autoplay="true" data-dots="true">
            <!-- 슬라이드 이미지들 -->
        </div>
        ```
    *   **동적 데이터 바인딩**: 서버에서 렌더링된 데이터(예: 상품 ID, 사용자 ID)를 요소에 저장해두고, 사용자의 상호작용(예: '삭제' 또는 '장바구니 담기' 클릭) 시 JavaScript가 이 값을 참조하여 AJAX 요청을 보냅니다.
        ```html
        <ul>
            <li data-item-id="101">상품 A <button data-action="delete">삭제</button></li>
            <li data-item-id="102">상품 B <button data-action="delete">삭제</button></li>
        </ul>
        <script>
            document.querySelector('ul').addEventListener('click', function(e) {
                if (e.target.dataset.action === 'delete') {
                    const listItem = e.target.closest('li');
                    const itemId = listItem.dataset.itemId;
                    console.log(`서버에 ${itemId} 삭제 요청을 보냅니다.`);
                    // fetch(`/api/items/${itemId}`, { method: 'DELETE' });
                }
            });
        </script>
        ```

-   **2.8.6. 모범 사례 및 주의점 (Best Practices & Caveats)**
    *   **데이터는 항상 문자열(String)**: `data-*` 속성으로 읽어온 값은 항상 문자열입니다. 숫자나 불리언 값으로 사용하려면 JavaScript에서 명시적으로 변환해야 합니다.
        ```javascript
        const item = document.getElementById('item');
        // item.dataset.price는 '5000' (string)
        const price = parseInt(item.dataset.price, 10);
        // item.dataset.isAvailable는 'true' (string)
        const isAvailable = item.dataset.isAvailable === 'true';
        ```
    *   **접근성 및 SEO**: `data-*` 속성의 내용은 스크린 리더나 검색 엔진 크롤러가 읽지 않습니다. 사용자나 검색 엔진에게 중요한 정보는 이 속성에 저장해서는 안 됩니다. 반드시 일반 텍스트, `alt` 속성, `aria-*` 속성 등을 사용해야 합니다.
    *   **저장 데이터의 크기**: `data-*`는 간단한 메타데이터나 상태 값을 저장하기 위한 것입니다. 크기가 큰 데이터나 복잡한 JSON 구조를 문자열로 저장하는 것은 HTML 문서의 크기를 불필요하게 늘리고 성능에 영향을 줄 수 있으므로 피해야 합니다.
    *   **스타일링 vs 상태**: 요소의 상태를 나타낼 때 `data-state="active"`와 같은 `data-*` 속성을 사용하는 것과 `.is-active` 같은 클래스를 사용하는 것 사이에는 미묘한 차이가 있습니다.
        *   **클래스**: 주로 스타일링 그룹을 정의하는 데 사용됩니다. (예: `.btn`, `.btn-primary`)
        *   **`data-*` 속성**: JavaScript가 사용하는 값(상태, 식별자 등)을 가질 때 더 의미론적입니다. CSS는 이 상태 값을 기반으로 스타일을 적용할 수 있습니다. 둘 다 유효하지만, `data-*`는 "상태"라는 의미를 더 명확하게 전달할 수 있습니다.
